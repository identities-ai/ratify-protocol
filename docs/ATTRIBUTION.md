# Attribution & Badge Program

Ratify Protocol is open source — Apache 2.0 for code, CC-BY-4.0 for the spec.
Use it commercially, build on it, modify it. No permission needed.

When you ship a product built on Ratify Protocol, we ask — but don't require —
that you say so. The reason: your users' trust in "verified" rests on a
cryptographic protocol, not your marketing. Making that visible lets them check
it independently. It's the same reason apps display "Messages secured with the
Signal Protocol."

## What we ask

**In your docs or security page:**

> Agent authorization powered by [Ratify Protocol](https://identities.ai/protocol)

**In your product UI** (next to verification results):

| Asset | Use when |
|---|---|
| [`assets/badge-verified-dark.svg`](../assets/badge-verified-dark.svg) | Dark or colored backgrounds |
| [`assets/badge-verified-light.svg`](../assets/badge-verified-light.svg) | White or light backgrounds |
| [`assets/badge-powered-dark.svg`](../assets/badge-powered-dark.svg) | Footer / "built with" placement (dark) |
| [`assets/badge-powered-light.svg`](../assets/badge-powered-light.svg) | Footer / "built with" placement (light) |

Link any badge to `https://identities.ai/protocol`. Don't alter the badge
text, colors, or proportions, and don't use it to imply endorsement.
